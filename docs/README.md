# 📋 기능목록

## 예약 일자 - Reserved Date

- [ ]  ✅ 날짜를 통해 `평일 할인`에 해당하는지 확인한다.
- [ ]  ✅ 날짜를 통해 `주말할인`에 해당하는지 확인한다.
- [ ]  ✅ 날짜를 통해 `특별 할인`에 해당되는지 확인한다.
- [ ]  ✅ 날짜를 통해 `크리스마스 디데이 할인`에 해당되는지 확인한다.
- [ ]  ✅ `숫자가 들어왔는지 검증한다`를 통해 예약 일자를 검증한다
- [ ]  ✅ 날짜가 `1`에서 `31` 사이의 정수값인지 검증한다.

## 예약 메뉴 - Reserved Menus Group
- [ ]  ✅ 예약하고자 하는 메뉴를 추가할 수 있다.
- [ ]  ✅ `,`로 예약 메뉴별로 나눌 수 있다.
- [ ]  ✅ `-`로 메뉴 이름과 수량을 나눌 수 있다.
- [ ]  ✅ 예약한 메뉴이름과 예약한 메뉴의 수량을 받을 수 있다
- [ ]  ✅ 예약한 메뉴중 디저트는 몇개가 있는지 확인한다
- [ ]  ✅ 예약한 메뉴중 메인 메뉴는 몇개가 있는지 확인한다

## 예약 메뉴 검증 - Reserved Menu Group Validator
- [ ]  ✅ 음료만 주문하였는지 검증한다
- [ ]  ✅ 올바른 메뉴명으로 주문하였는지 검증한다.
- [ ]  ✅ 주문한 메뉴의 수가 올바른 범위안에 존재하는지 검증한다
- [ ]  ✅ 메뉴의 입력 형식이 올바른지 검증한다.
- [ ]  ✅ 중복 메뉴가 입력되었는지 검증한다.

## 계산기 - Calculator
- [ ]  ✅ 메뉴의 할인 전 금액을 구한다
- [ ]  ✅ `크리스마스 디데이 할인` 으로 할인 받을 수 있는 금액을 구한다.
- [ ]  ✅ `평일 할인`으로 할인받을 수 있는 금액을 구한다.
- [ ]  ✅ `주말 할인`으로 할인 받을 수 있는 금액을 구한다.
- [ ]  ✅ `특별 할인`으로 할인 받을 수 있는  금액을 구한다.
- [ ]  ✅ `증정 이벤트`로 혜택받을 수 있는 금액을 구한다.
- [ ]  ✅ `할인 후 예상 결제 금액 = 할인 전 총주문 금액 - 할인 금액` 수식 을 통해 할인 후 예상 결제 금액을 구할 수 있다.
- [ ]  ✅ `총혜택 금액 = 할인 금액의 합계 + 증정 메뉴의 가격` 수식 을 통해 총 혜택 금액을 구할 수 있다.

## 예약 주문 - Reserved Order

- [ ]  ✅ 할인을 받을 수 있는 최소 주문 금액을 넘었는지 확인한다
- [ ]  ✅ 받을수 있는 할인들의 목록을 구한다.
- [ ]  ✅ 혜택 내역을 건네줄수 있다.
- [ ]  ✅ 계산기를 통해 `총 혜택 금액`을 구할 수 있다.
- [ ]  ✅ 계산기를 통해 `할인 후 예상 결제 금액`을 구할 수 있다.
- [ ]  ✅ 총 혜택 금액에 따른 12월 이벤트 배지 부여한다.
    - 5000원 이상 1만원 미만은 별 배지
    - 1만원 이상은 2만원 미만은 트리 배지
    - 2만원 이상은 산타 배지

## 입력 기능 - InputView
- [ ]  ✅ 12월 중 식당 예상 방문 날짜를 입력받을 수 있다.
- [ ]  ✅ 주문할 메뉴와 개수를 입력받을 수 있다.
- [ ]  ✅ 예외 발생시 해당 부분부터 다시 입력을 받을 수 있다.

## 출력 기능 - OutPutView
- [ ]  ✅ 사용자를 환영하는 문구를 출력한다
- [ ]  ✅ 주문 메뉴를 출력한다.
- [ ]  ✅ 할인 전 총 주문 금액을 출력한다
- [ ]  ✅ 증정 메뉴에 대해 출력한다.
- [ ]  ✅ 고객에게 적용된 이벤트 내역을 출력해준다.
- [ ]  ✅ 총혜택 금액을 출력한다
- [ ]  ✅ 할인후 예상 결제 금액을 출력한다
- [ ]  ✅ 이벤트 배지를 출력한다
- [ ]  ✅ 금액은 `1,000`원, `1,000,000`원 과 같이 표현한다

## 공통 검증 - CommonValidator
- [ ]  ✅ 정수가 들어왔는지 검증한다
- [ ]  ✅ 공백이 들어왔는지 검증한다
- [ ]  ✅ 빈칸이 존재하는지 검증한다

## 변환 로직 - Utility
- [ ]  ✅ 문자를 숫자로 바꾼다
